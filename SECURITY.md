# Security Policy

## Versoes suportadas

Este projeto nao segue suporte retroativo por linha de versao.

Hoje, a unica versao suportada para correcao de seguranca e a branch ativa `main`.
Snapshots antigos, forks locais e builds fora da `main` nao entram no ciclo de correcao.

| Branch | Suporte |
| --- | --- |
| `main` | Sim |
| snapshots antigos | Nao |
| forks nao oficiais | Nao |

## Como reportar uma vulnerabilidade

Nao abra falha de seguranca em issue publica.

Envie o reporte com:

- descricao objetiva do problema
- impacto real
- passos para reproduzir
- ambiente afetado
- evidencias disponiveis

Canal atual de resposta:

- GitHub Security Advisory privado, quando disponivel no repositorio
- ou contato direto com o mantenedor do projeto

## O que esperar

- triagem inicial assim que o reporte for revisado
- confirmacao se a falha foi reproduzida ou nao
- correcao na `main` quando o problema for valido
- divulgacao publica so depois que a correcao estiver pronta, quando fizer sentido

Se o reporte nao tiver dados suficientes para reproducao, ele pode ficar pendente ate chegar evidencia melhor.
